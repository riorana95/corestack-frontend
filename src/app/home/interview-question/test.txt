Main list cover kar lete hain jo Angular projects mein real-life mein sabse zyada use hote hain:
Filtering Operators (The Gatekeepers)

* filter: Yeh sirf unhi values ko aage jane deta hai jo condition meet karein (jaise if condition).
* take(n): Yeh stream se sirf pehli n values leta hai aur phir stream band (complete) kar deta hai.
* distinctUntilChanged: Yeh tabhi data bhejta hai jab value change ho; agar purani aur nayi value same hai toh block kar deta hai.
* debounceTime(ms): Yeh thodi der wait karta hai; agar user ne typing rok di, tabhi aage data bhejega (Search bar optimization).

Combination Operators (The Mixers)

* combineLatest: Jab bhi kisi ek stream mein change hota hai, yeh saari streams ki latest values ko milakar ek array mein bhej deta hai.
* withLatestFrom: Yeh primary stream ka wait karta hai aur doosri stream se sirf "latest value" utha kar saath chipka deta hai.
* forkJoin: Yeh sabka khatam hone ka wait karta hai aur aakhri mein ek saath result deta hai (Jaise Promise.all).

Utility & Error Handling

* tap: Yeh sirf "dekhne" ke liye hai (side-effects); data ko change nahi karta, bas logging ya loading spinner off karne ke kaam aata.
* catchError: Agar stream mein error aaye, toh yeh use catch karke app ko crash hone se bachata hai.
* of: Yeh kisi bhi simple value (string, number, array) ko Observable mein convert kar deta hai.
* from: Yeh kisi Array ya Promise ko Observable mein convert kar deta hai.

Special Subject

* AsyncSubject: Yeh tab tak kuch nahi bolta jab tak stream complete na ho jaye; complete hote hi sirf aakhri value bhejta hai.

RxJS ek aisi library hai jo data ko ek behti hui nadi (Stream) ki tarah handle karti hai. Iska kaam hai asynchronous events (click, API calls) ko line mein laga kar filter ya transform karna.
Yahan list ki simple definitions hain:
Mapping Operators (The Traffic Controllers)

* switchMap: Purani request ko cancel kar deta hai aur sirf latest wali pe dhyan deta hai (Best for Search bars).
* mergeMap: Saari requests ko ek saath parallel chalne deta hai, kisi ka wait nahi karta.
* concatMap: Ek-ek karke queue mein chalata hai; jab pehli khatam hogi tabhi doosri shuru hogi.

Subjects (The Broadcasters)

* Subject: Ek loudspeaker ki tarah hai; jo sun raha hai usse sirf current awaaz sunayi degi, purani nahi.
* BehaviorSubject: Iske paas humesha ek value rehti hai; naya banda join karega toh usse turant "last value" mil jayegi.
* ReplaySubject: Yeh history yaad rakhta hai; naye bande ko purani saari (ya fix number of) values sunata hai.

The Basics

* Promise: Ek hi baar data deta hai aur handle karna mushkil hai (One-shot).
* Async/Await: Promise ko likhne ka saaf-suthra tareeka hai taaki code synchronous dikhe.

RxJS is a powerful library for reactive programming that uses Observables to handle asynchronous data streams. Think of it as a set of tools to manage and transform events (like clicks, timers, or API responses) as they happen over time. 

The Core Mechanism
Observable: A blueprint for a stream of data that emits values over time.
Pipe: A method used to chain operators together, creating a step-by-step processing pipeline for your data.
Subscription: The act of "listening" to an Observable. Data only starts flowing once you subscribe. 

Essential RxJS Operators
Operators are categorized by how they handle the data stream. 
1. Creation Operators (Starting the Stream)
These functions create new Observables from scratch. 

of(): Emits a simple sequence of values (like numbers or an array) and then completes.
from(): Converts an existing array, promise, or iterable into a stream of individual values.
interval(ms): Emits a sequential number every set number of milliseconds. 

2. Transformation Operators (Changing the Data)
These modify the values as they pass through the pipe. 

map(): Transforms each value (e.g., doubling a number or extracting a property from an object).
switchMap(): Cancels any previous pending request and switches to the newest one. Ideal for search bars to avoid "race conditions".

-------------------------------------------------
import { FormControl } from '@angular/forms';
import { debounceTime, distinctUntilChanged, switchMap, tap } from 'rxjs/operators';

// 1. Input control banaya
searchControl = new FormControl();
loading = false;

ngOnInit() {
  this.searchControl.valueChanges.pipe(
    // A. Thoda ruko! (Wait for 300ms taaki har ek key-press pe API na jaye)
    debounceTime(300),

    // B. Same cheez dubara mat bhejo! (Agar user ne 'A' likha fir delete karke 'A' likha)
    distinctUntilChanged(),

    // C. Side effect! (Sirf loading spinner dikhane ke liye)
    tap(() => this.loading = true),

    // D. MAGIC! Purani API call cancel karo aur sirf latest wali ka data lo
    switchMap(searchterm => this.myService.searchAPI(searchterm))

  ).subscribe(results => {
    this.results = results;
    this.loading = false;
    console.log('Final Results:', results);
  });
}

// Ek-ek karke save karega (Sequential)

  concatMap(data => this.api.save(data))
).subscribe();

3. Filtering Operators (Selecting What Matters)
These decide which values are allowed to continue down the pipe. 

filter(): Only lets values pass that satisfy a specific condition.
debounceTime(ms): Ignores values that come too fast; it only emits if a certain time passes without a new value.
distinctUntilChanged(): Prevents duplicate consecutive values from being emitted.
take(n): Only takes the first n values and then automatically unsubscribes. 

4. Combination & Error Handling
combineLatest(): Joins multiple streams and emits a value whenever any of them changes, using the latest from all.
forkJoin(): Waits for all streams to complete and then emits their final values as an array (like Promise.all).
-----------------------------------

// Profile aur Settings dono aa jayengi tabhi screen dikhegi
forkJoin({
  profile: this.api.getProfile(),
  settings: this.api.getSettings()
}).subscribe(res => console.log(res.profile, res.settings));

catchError(): Catches errors in the stream and allows you to return a fallback value so the app doesn't crash.
tap(): Used for "side effects" like logging or showing a spinner; it doesn't change the data itself.

BehaviorSubject
BehaviorSubject ek aisa "smart broadcaster" hai jo hamesha latest value ko yaad rakhta hai.
Iska sabse bada fayda ye hai ki agar koi naya subscriber (listener) baad mein join kare, toh usse purana wait nahi karna padta—usse turant current value mil jati hai.
BehaviorSubject ki 3 Khaas Baatein:
Initial Value: Isko shuru karne ke liye ek value deni padti hai (Jaise: 0 ya null).
Current Value Yaad Rakhta Hai: Ye hamesha apne paas "last emitted value" store karta hai.
Instant Delivery: Naye subscriber ko subscribe karte hi turant latest data milta hai.

Real-Life Example: Radio vs. Recording
Normal Subject: Ek Live Radio ki tarah hai. Agar aapne 10 baje radio on kiya, toh 9 baje wala gaana aap miss kar chuke ho.
BehaviorSubject: Ek WhatsApp Status ki tarah hai. Status 2 ghante pehle dala ho, par jab bhi aap dekhoge (subscribe karoge), aapko latest status dikhayi dega.

import { BehaviorSubject } from 'rxjs';

// Initial value 'Default' ke saath start kiya
const myBS = new BehaviorSubject<string>("Default");

myBS.subscribe(val => console.log("Sub A:", val));
// Output: Sub A: Default (Turant mil gayi!)

myBS.next("New Value");
// Output: Sub A: New Value

myBS.subscribe(val => console.log("Sub B:", val));
// Output: Sub B: New Value (Naye wale ko bhi latest value mil gayi)

For an interview, you can define Angular’s architecture as a Component-Based Modular Framework. It follows a tree-like structure where everything is organized into blocks.
Here are the 5 Core Pillars of Angular Architecture:
1. Modules (NgModule)
The container for your app. It groups related components, directives, and services together.
Root Module: Every app has at least one (AppModule).
Feature Modules: Used to split the app into smaller, manageable chunks (e.g., UserModule, AdminModule).
2. Components (The UI Logic)
The basic building block of the UI. Every component consists of:
Template: The HTML (What the user sees).
Styles: The CSS (How it looks).
Class: The TypeScript logic (How it behaves).
Metadata: Defined via the @Component decorator.
3. Templates & Data Binding
How the UI and Logic talk to each other:
Interpolation {{ }}: Logic to UI.
Property Binding [ ]: Passing values to elements.
Event Binding ( ): User actions (clicks) to Logic.
Two-way Binding [( )]: Syncing both (using ngModel).
4. Services & Dependency Injection (DI)
Services: Classes that hold reusable business logic (like API calls) that shouldn't live in components.
DI: The system that "injects" these services into components whenever they are needed.
5. Directives & Pipes
Directives: Transform the DOM (e.g., *ngIf for logic, *ngFor for loops).
Pipes: Transform data in the UI (e.g., | date, | uppercase, | async).
Interview "Pro Tip" Summary:
"Angular architecture is a hierarchy of Components organized into Modules. It uses Services to share logic via Dependency Injection, and uses Templates with Data Binding to create a dynamic user experience."
